##　不明点・詰まった点の報告方法

1. **リポジトリに移動する**

   - GitHub アカウントにログインし、対象のリポジトリを開きます。
   - 左上にある「Code」「Issues」などのタブが表示されているはずです。

2. **Issues タブを選択**

   - 画面上部の「Issues」タブをクリックして、Issue ページを開きます。

3. **新しい Issue を作成**

   - 「New issue」ボタン（緑色）をクリックします。

4. **テンプレートを選択**

   - Issue テンプレートが設定されている場合、「詰まった箇所の記録」などのテンプレートを選択します。
   - テンプレートが表示されない場合、空の Issue から作成するか、リポジトリの管理者にテンプレート設定を依頼してください。

5. **タイトルと説明を記入**

   - タイトルには簡潔に問題の概要を記載します（例: 「React コンポーネントのデータ渡しエラー」）。
   - 説明欄に以下の内容を具体的に記載します。
     - **詰まった箇所**: 問題が発生した箇所や詳細な内容。
     - **試したこと**: 解決のために試した内容や調査結果。
     - **次にすること**: 予定している次のステップ（例: ドキュメント確認、メンターへの質問など）。

6. **ラベルや担当者を設定**

   - 必要に応じて右側でラベルや担当者（Assignees）を設定します。
   - ラベル例: `bug`, `help wanted`, `question` など。

7. **Issue の作成**
   - 入力が完了したら、画面下部の「Create」ボタンをクリックして Issue を作成します。

## 留意点

1. **進捗管理**:
   - 不明点や詰まった箇所を簡単にメモする習慣をつける。
2. **LLM の使い方**:
   - **フェーズ 1**: 使用禁止（自力で解決力を養う）
   - **フェーズ 2 以降**: 使用可（ただし、生成されたコードを完全に理解すること）

## よくある Q&A

### Q1. どれくらい時間がかかりますか？

- **目安**: 約 2〜4 ヶ月で基礎から実践まで進め、実務に参加できるスキルを目指します。

### Q2. エラーが出て解決できません。どうすればいいですか？

- **解決方法**:
  1. エラー文を検索する。
  2. フェーズ 2 以降で LLM を活用する。
  3. メンターに質問する際は、エラー内容を簡潔に伝える。

### Q3. 書籍は買うべきですか？

- **判断基準**:
  - **おすすめ**: 普遍的な知識（コンピューター構造、ネットワーク、設計）
  - **おすすめしない**: 言語やフレームワークなど（情報が古くなるため）

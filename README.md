## はじめに

### **各フェーズの終了目安**

| フェーズ       | 終了目安             | 最低値 | 最高値 |
| -------------- | -------------------- | ------ | ------ |
| **基礎学習**   | 開始後 **1〜2 週間** | 7 日   | 14 日  |
| **応用学習**   | 開始後 **1〜2 週間** | 7 日   | 14 日  |
| **Git/GitHub** | 開始後 **1〜2 週間** | 7 日   | 14 日  |
| **実践**       | 開始後 **4〜8 週間** | 28 日  | 56 日  |
| **チーム開発** | 開始後 **1〜2 週間** | 7 日   | 14 日  |

## 不明点・詰まった点の報告方法

1. **リポジトリに移動する**

   - GitHub アカウントにログインし、対象のリポジトリを開きます。
   - [リンク](https://github.com/dotlog-inc/dot-input)
   - 左上にある「Code」「Issues」などのタブが表示されているはずです。

2. **Issues タブを選択**

   - 画面上部の「Issues」タブをクリックして、Issue ページを開きます。
   - <img width="1800" alt="スクリーンショット 2025-01-27 20 43 59" src="https://github.com/user-attachments/assets/36bcf2d2-becc-48a6-9d7c-031047e249b2" />


3. **新しい Issue を作成**

   - 「New issue」ボタン（緑色）をクリックします。
   - <img width="1800" alt="スクリーンショット 2025-01-27 20 44 26" src="https://github.com/user-attachments/assets/e8fc01be-efe4-4f92-b208-4bdc80c9627b" />


4. **テンプレートを選択**

   - Issue テンプレートが設定されている場合、「詰まった箇所の記録」などのテンプレートを選択します。
   - テンプレートが表示されない場合、空の Issue から作成するか、リポジトリの管理者にテンプレート設定を依頼してください。
   - <img width="1800" alt="スクリーンショット 2025-01-27 20 44 46" src="https://github.com/user-attachments/assets/f1e2d568-9afa-4268-a4fd-40ec49f24d38" />


5. **タイトルと説明を記入**

   - タイトルには簡潔に問題の概要を記載します（例: 「React コンポーネントのデータ渡しエラー」）。
   - 説明欄に以下の内容を具体的に記載します。
     - **詰まった箇所**: 問題が発生した箇所や詳細な内容。
     - **試したこと**: 解決のために試した内容や調査結果。
   - <img width="1800" alt="スクリーンショット 2025-01-27 20 45 15" src="https://github.com/user-attachments/assets/dc628333-2698-4b32-a743-d8ddcd79e2a7" />

7. **Issue の作成**
   - 入力が完了したら、画面下部の「Create」ボタンをクリックして Issue を作成します。
  
8. **コメントの確認**
   - 担当者からのコメントを確認しましょう。
   - わからなければコメントを担当者をアットマーク（＠）でメンションして返信しましょう。
   - <img width="1800" alt="スクリーンショット 2025-01-27 20 59 33" src="https://github.com/user-attachments/assets/3f1c6b8b-0a04-4c81-b8c8-703ef91c09e7" />

9. **完了**
   - 解決したら、「close」ボタンを押しましょう。
   - <img width="1800" alt="スクリーンショット 2025-01-27 20 58 50" src="https://github.com/user-attachments/assets/ecf3b505-a8d4-459e-9059-a2d9643f91f8" />

## よくある Q&A

### Q1. どれくらい時間がかかりますか？

- **目安**: 約 2〜4 ヶ月で基礎から実践まで進め、実務に参加できるスキルを目指します。

### Q2. エラーが出て解決できません。どうすればいいですか？

- **解決方法**:
  1. エラー文を検索する。
  2. フェーズ 2 以降で LLM を活用する。
  3. メンターに質問する際は、[不明点・詰まった点の報告方法](#不明点・詰まった点の報告方法)を使って報告をする。

### Q3. 書籍は買うべきですか？

- **判断基準**:
  - **おすすめ**: 普遍的な知識（コンピューター構造、ネットワーク、設計）
  - **おすすめしない**: 言語やフレームワークなど（情報が古くなるため）
 
### Q4. LLM（大規模言語モデル）はどのように使えばよいですか？
- **フェーズ1（基礎学習）**:  
  - **使用しない**: エラー解決力やリサーチスキルを養うことが目的です。自分で解決する力を身につけましょう。
  
- **フェーズ2（応用学習）以降**:  
  1. **バグ解決の補助**: エラー文やコードの改善方法を尋ねる際に使用してよい。
  2. **コード生成**: プロジェクト内のコードを生成する際に活用可能。ただし、以下を守ること：
     - 出力されたコードを**必ず理解する**。
     - **理解とは**: コードがどのような機能を実現し、なぜそのように動くのかを詳細に説明できる状態。
  3. **学習リソースの補完**: 概念の説明や疑問点の簡単な解説に活用する。

- **注意点**:
  - **鵜呑みにしない**: 出力結果が誤っている場合があるため、自分で検証する。
  - **質問力を磨く**: 明確かつ具体的な質問をすることで、有用な回答が得られやすくなります。
